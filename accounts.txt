                    ACCOUNT LIST - WellCo System
═══════════════════════════════════════════════════════════════════════════════

PURPOSE:
--------
This file stores all user account information for the WellCo application.
It serves as a backup and synchronization point between SQL database and
in-memory (ArrayList) storage. The file is automatically updated whenever
accounts are created, modified, or deleted through the application.

FILE FORMAT:
-----------
The file uses a pipe-separated (|) format for readability in Notepad.
Each account line contains the following fields in order:
  1. ROLE           - User role (Customer, Manager, or Admin)
  2. USERNAME       - Unique username for login
  3. PASSWORD       - User password (stored in plain text for backup)
  4. EMAIL          - User email address
  5. CREATED DATE    - Account creation timestamp (yyyy-MM-dd HH:mm:ss)
  6. ORIGINAL USERNAME - Original username (for account history)

ALTERNATIVE FORMAT (for manual editing):
---------------------------------------
You can also use comma-separated format when manually editing:
  ROLE, USERNAME, Password, Email, CreatedDate, OriginalUsername
Example: Customer, john_doe, password123, john@email.com, 2025-01-15 10:30:00, john_doe

IMPORTANT NOTES:
---------------
- This file is automatically generated and updated by the application.
- Manual edits are supported but will be overwritten on next update.
- Accounts are loaded from this file on application startup.
- Changes made through the application UI will update this file.
- The file serves as a backup when SQL database is unavailable.
